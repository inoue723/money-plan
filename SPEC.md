- 版数: 1.1(ドラフト)
- 作成日: 2026-07-11
- ステータス: レビュー待ち

---

## 1. 概要

### 1.1 目的

ユーザーが自身の収入・支出・家族構成などの情報を入力することで、将来の資産推移をシミュレーションし、時系列グラフで可視化するWebシステムを提供する。ライフプランニング(住宅購入、教育費、老後資金など)の意思決定を支援することを目的とする。

### 1.2 コンセプト

- **簡単入力**: 年収・家賃・生活費など最低限の項目だけで即座にシミュレーション可能
- **詳細な内訳**: 税金・社会保険料・投資利益などの内訳を年単位で確認できる透明性
- **視覚的な理解**: 収入・支出・資産残高の推移をインタラクティブなグラフで表示

### 1.3 想定ユーザー

- 将来の資産形成を計画したい個人(20〜50代の給与所得者を主対象)
- 結婚・出産・住宅購入などのライフイベントを控え、資金計画を立てたい世帯

### 1.4 注意事項(免責)

本システムは簡易シミュレーションであり、金融商品の勧誘や投資助言を行うものではない。税制・社会保険制度は簡略化したモデルで計算し、実際の金額とは異なる場合がある旨を画面上に明示する。

#### F-01 基本情報入力

| 項目 | 型 | 必須 | 備考 |

| 現在の投資資産額 | 金額(万円) | - | デフォルト 0 |

| 項目 | 型 | 必須 | 備考 |

| 年金受給額(年額) | 金額(万円) | - | デフォルトは年収から概算した目安値を提示 |
| 配偶者の年収 | 金額(万円) | - | 配偶者ありの場合。税計算は本人と同様に実施 |
| その他の収入(年額) | 金額(万円) | - | 副業・不動産収入など(手取り額として扱う) |

#### F-03 支出情報入力

| 項目 | 型 | 必須 | 備考 |
|------|-----|------|------|
| 家賃(月額) | 金額(万円) | ○ | 持ち家の場合は 0 とし、住宅ローンをライフイベントで設定 |
| 生活費(月額) | 金額(万円) | ○ | 食費・水道光熱費・通信費などの合計 |
| 保険料(月額) | 金額(万円) | - | 生命保険・医療保険など |
| その他固定費(月額) | 金額(万円) | - | サブスクリプション、駐車場代など |
| 物価上昇率 | %(年率) | - | デフォルト 1.0%。生活費・家賃に適用 |

#### F-04 ライフイベント設定

以下のイベントを任意の年に複数登録できる。

| イベント種別 | パラメータ | 支出への影響 |
|--------------|-----------|--------------|
| 結婚 | 発生年齢、費用 | 一時費用 + 以降の生活費係数変更 |
| 出産・子ども誕生 | 発生年齢 | 教育費モデルを自動適用(後述)、児童手当を加算 |
| 住宅購入 | 発生年齢、物件価格、頭金、ローン金利、返済期間 | 頭金を一時支出、以降ローン返済を年次支出に計上、家賃は 0 に |
| 車購入 | 発生年齢、価格、買替周期 | 周期ごとに一時支出 + 維持費(年額)を計上 |
| その他一時支出 | 発生年齢、名称、金額 | 指定年に一括計上(例: 旅行、リフォーム) |
| その他一時収入 | 発生年齢、名称、金額 | 指定年に一括計上(例: 相続、贈与) |

投資は「投資枠(口座)」ごとに設定する。投資枠は名義(本人 / 配偶者)× 種別(NISA / 特定口座 /
iDeCo / 小規模企業共済)の組み合わせで追加し、同じ組み合わせは 1 枠まで。各枠は積立・利回り・
取り崩しを独立に持つ。

| 項目 | 型 | 必須 | 備考 |
|------|-----|------|------|
| 口座種別 | NISA / 特定口座 / iDeCo / 小規模企業共済 | ○ | 非課税・課税・拠出全額所得控除などの税制を切り替え |
| 積立設定(複数) | 月額積立(年齢期間ごとの月額) / 一括投資(年齢・金額) | - | 1 枠に複数登録可。年齢によって積立額を変えられる(例: 40〜45歳は月5万、46〜50歳は月1万)。特定年齢の一括投資(例: 40歳で1000万)にも対応 |
| 想定利回り | %(年率) | - | デフォルト 3.0%。0〜15% の範囲 |
| 取り崩し設定(複数) | 分割取崩(期間) / 一括取崩(年齢・金額) | - | 老後の投資資産の取り崩しに対応 |

- NISA枠は非課税枠内の運用益を非課税として計算し、投資上限(生涯1800万・年間360万)を名義ごとに適用する。

### 2.3 シミュレーション機能(F-06)

#### 2.3.1 計算の基本方針

- 計算単位は**年次**とする(現在年齢から終了年齢まで1年刻み)。
- 各年について以下を計算する:

```
手取り収入 = 額面収入 − 所得税 − 住民税 − 社会保険料
年間収支   = 手取り収入 + その他収入 − 年間支出 − 投資積立額
預金残高   = 前年の預金残高 + 年間収支 (+ 投資取崩額)
投資資産   = (前年の投資資産 + 積立額) × (1 + 利回り) − 取崩額
総資産     = 預金残高 + 投資資産
```

#### 2.3.2 税金・社会保険料の計算モデル

給与所得者を前提とした簡易モデルで計算する(計算年の税制は2026年度を基準とし、定数は設定ファイルで管理して改正に追随できる構造とする)。

1. **給与所得控除**: 年収に応じた速算表で計算
2. **社会保険料**: 健康保険・厚生年金・雇用保険の料率合計(概算 約15%)を給与に乗じて計算(上限あり)
3. **所得税**: 課税所得(給与所得 − 社会保険料控除 − 基礎控除 − 配偶者控除 − 扶養控除)に超過累進税率を適用。復興特別所得税を含む
4. **住民税**: 課税所得 × 10% + 均等割(地域差はv1では考慮しない)
5. **児童手当**: 子どもの年齢に応じて収入に加算
6. **年金受給**: 受給開始年齢以降は公的年金等控除を適用した簡易計算
7. **投資利益への課税**: 課税口座は運用益に対し20.315%(取崩時に課税)、NISA枠は非課税

#### 2.3.3 教育費モデル

子ども1人あたり、年齢に応じた標準教育費を自動計上する。進路(公立/私立)は子どもごとに選択可能。

| 区分 | 公立(年額目安) | 私立(年額目安) |
|------|----------------|----------------|
| 未就学(0〜5歳) | 30万円 | 50万円 |
| 小学校 | 35万円 | 170万円 |
| 中学校 | 55万円 | 145万円 |
| 高校 | 50万円 | 100万円 |
| 大学 | 250万円(国公立) | 400万円(私立文系)/ 550万円(私立理系) |

※金額は文部科学省の調査等に基づく目安値。設定で上書き可能とする。

#### 2.3.4 計算結果データ

年ごとに以下の項目を算出し、出力に使用する。

- 本人年齢・西暦・家族の年齢
- 収入内訳: 額面給与、配偶者給与、年金、児童手当、その他収入、投資利益(評価益)
- 控除内訳: 所得税、住民税、社会保険料
- 支出内訳: 家賃/住宅ローン、生活費、教育費、保険料、その他固定費、ライフイベント費用
- 資産: 年間収支、預金残高、投資資産評価額、総資産

### 2.4 出力機能

#### F-07 グラフ表示

1. **資産推移グラフ(メイン)**
   - 横軸: 年(西暦および本人年齢を併記)
   - 縦軸: 金額(万円)
   - 系列: 預金残高(棒または面)、投資資産(積み上げ)、総資産(折れ線)
   - 資産がマイナスになる年は警告色で強調表示
2. **収支グラフ**
   - 年ごとの収入(正方向の積み上げ棒: 手取り給与・年金・児童手当・その他)と支出(負方向の積み上げ棒: 住居費・生活費・教育費・税金等)を対比表示
   - 年間収支(収入−支出)を折れ線で重ねる
3. **共通仕様**
   - グラフ上のホバーで該当年の主要数値をツールチップ表示
   - グラフの任意の年をクリックすると、その年の詳細内訳(F-08)へ遷移
   - ライフイベント発生年にはマーカーとラベルを表示

#### F-08 年次内訳表示

- 選択した年について、以下をカテゴリ別に表形式で表示する:
  - 収入の内訳(額面給与、手取り、年金、児童手当、投資利益 など)
  - 税金・社会保険料の内訳(所得税、住民税、健康保険、厚生年金、雇用保険)
  - 支出の内訳(住居費、生活費、教育費、保険、イベント費用)
  - 資産の状況(年間収支、預金残高、投資資産、総資産、前年比)
- 全期間の年次テーブル(一覧表)も提供し、CSVダウンロードを可能とする

#### F-09 プラン保存・読込

- 入力条件一式を「プラン」として名前を付けて保存できる
- v1では未認証利用を前提とし、ブラウザのローカルストレージに保存する
- 将来的にユーザー認証(F-11)導入後はサーバー側に保存する

### 3.2 レイアウト・UX方針

- 入力と結果を **1画面に統合**する。左側のサイドパネルに入力フォーム、右側に資産推移グラフ・収支グラフを配置する
- 入力フォームは「基本情報 / 収入 / 支出 / ライフイベント / 投資」のセクションに分け、アコーディオン形式で開閉できる。必須項目のみでも結果を表示できる
- 入力値の**変更は即座に再計算され、右側のグラフへリアルタイムに反映**される
- 対象デバイスは **PCのみ**とし、レスポンシブ対応は行わない(想定最小画面幅: 1280px)

## 4. システム構成・技術スタック(案)

### 4.1 構成方針

シミュレーション計算は入力データのみに依存する純粋な計算であるため、**v1はフロントエンド完結型(SPA)** とし、サーバーサイドは静的ホスティングのみとする。これにより個人の金融情報を外部送信せずに利用できる(プライバシー面の利点として画面に明示)。

### 4.2 技術スタック

| 領域 | 技術 | 備考 |
|------|------|------|
| リポジトリ構成 | pnpm workspace(monorepo) | 将来のAPIサーバー追加等を見据えた構成 |
| フロントエンド | React + TypeScript(Vite) | |
| グラフ描画 | nivo(@nivo/bar, @nivo/line) | 積み上げ棒・折れ線・ツールチップに対応。描画パフォーマンスの実績を考慮して採用 |
| 状態管理 | Zustand(または React Context) | |
| スタイリング | Tailwind CSS | |
| 計算エンジン | TypeScript 純粋関数モジュール | UIから分離し単体テスト可能にする |
| データ保存 | localStorage | F-11 導入時に API + DB へ移行 |
| ホスティング | Cloudflare Pages | 静的ホスティング。F-11 導入時は Cloudflare Workers / D1 の利用を検討 |
| テスト | Vitest(計算エンジン)、Playwright(E2E) | |

### 4.3 モジュール構成(案)

今後の拡張(APIサーバー追加、計算エンジンの再利用など)を見据え、pnpm workspace による monorepo 構成とする。

```
.
├── pnpm-workspace.yaml
├── package.json               # ルート(共通スクリプト・開発ツール)
├── apps/
│   └── web/                   # Webアプリ(React + TypeScript + Vite)
│       └── src/
│           ├── features/      # 入力フォーム・グラフ等の画面機能
│           ├── components/    # 共通UIコンポーネント
│           └── stores/        # 状態管理・localStorage永続化
└── packages/
    └── finance-core/          # 計算エンジン(UI非依存パッケージ)
        └── src/
            ├── simulation.ts  # 年次シミュレーションの本体
            ├── tax.ts         # 所得税・住民税・社会保険料の計算
            ├── education.ts   # 教育費モデル
            ├── investment.ts  # 投資運用・取崩・課税の計算
            ├── constants/     # 税率表・教育費テーブル等(年度別に管理)
            └── types.ts       # 入出力の型定義
```

- `apps/web`: Webアプリ本体。将来 APIサーバー(例: `apps/api`)を追加する場合もこの階層に並べる
- `packages/finance-core`: 計算エンジンを独立パッケージとして切り出し、`apps/web` から依存する。将来サーバーサイドやCLIからも再利用可能とする

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | シミュレーション再計算は入力変更から 100ms 以内に完了し、グラフへ反映する |
| 対応環境 | PCブラウザのみ(Chrome / Edge / Safari / Firefox の最新版)。モバイル・タブレットは対象外 |
| アクセシビリティ | グラフには表形式の代替表示を用意。WCAG 2.1 AA 相当のコントラストを確保 |
| プライバシー | v1では入力データを外部送信しない。アナリティクスを導入する場合も入力値は収集しない |
| 保守性 | 税率・料率・教育費等の定数は年度別の設定ファイルに分離し、税制改正時に定数更新のみで対応可能とする |
| テスト | 計算エンジンは国税庁の計算例等を基準としたテストケースで検証する |

## 7. 用語定義

| 用語 | 定義 |
|------|------|
